AI Model & Inference Layer

This page explains how VISIONALTRIX turns raw F1 crash telemetry into an explainable safety signal for road-level ADAS. Use this walkthrough during demos to narrate the full stack—from data ingest to mitigation triggers.

End-to-End Pipeline

We follow a classic: ingest → preprocess → model → postprocess → serve.

1

Ingestion

Take F1 telemetry (speed, yaw, Gz, wheel speed, brake pressure), race logs, or sim outputs.

2

Preprocessing

Time-align signals, resample to 50–200Hz, normalize per channel, handle missing values.

3

Feature Engineering

Windowing, FFT over G-force, lateral vs longitudinal ratio, cornering signature.

4

Model Core

LSTM/Transformer autoencoder → reconstruction error → anomaly score (0–1).

5

Postprocessing

Thresholding, rule-based checks, attach mitigation actions.

6

Serving

Exposed via REST/WebSocket → shown in front end as “crash risk”.

Model Architecture

Sequence Autoencoder

Raw telemetry windows (T × F) are encoded with stacked LSTMs, compressed into a 32-d latent embedding, then reconstructed. Reconstruction error is calibrated into a crash-risk score.

  • • Encoder: 2 × LSTM layers (hidden 128)
  • • Latent bottleneck: Linear → 32 dims
  • • Decoder: 2 × LSTM + linear projection back to features
  • • Loss: Full sequence MSE on normalized signals
  • • Risk mapping: tanh(recon error) → {normal, elevated, high, critical}

Serving endpoints: /api/infer (POST) for live scoring and /api/model-metadata (GET) for UI sync.
